# 2026-05-01 10:21 - Added answer about MODBUS security and authentication models

Created a comprehensive answer page addressing the question: "Is there a security model for MODBUS in which a username and password protects accessing MODBUS registers?"

## Rationale

User asked specifically about username/password authentication for MODBUS register access. This is a common question from those familiar with modern web applications but new to industrial automation protocols. The answer required:

1. Clarifying that standard MODBUS has NO security features at all
2. Explaining that MODBUS/TCP Security uses X.509v3 certificates, not username/password
3. Describing role-based authorization for register access control
4. Contrasting the security models and explaining why certificates were chosen over passwords
5. Providing practical guidance for both secure and insecure deployments

The answer synthesizes information from multiple sources:
- MODBUS/TCP Security specification (mutual TLS authentication)
- MODBUS TCP implementation guide (optional IP-based access control)
- MODBUS Application Protocol specification (no built-in security)
- Existing wiki pages on TLS and MODBUS security
