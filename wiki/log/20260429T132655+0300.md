# 2026-04-29 13:26 - Converted Log from Single File to Directory Structure

## Changes Made

**Converted wiki/log.md to wiki/log/ directory structure:**

The project previously used a single `wiki/log.md` file to track all changes. This has been converted to a directory-based approach where each change is logged in a separate file.

### Rationale for Change

1. **Merge conflict prevention:** Multiple contributors can now work simultaneously without conflicts
2. **Better version control:** Individual files are easier to track in git history
3. **Scalability:** Easier to manage as the wiki grows
4. **Machine readability:** Structured YAML frontmatter enables automated analysis

### Implementation

1. **Created wiki/log/ directory** to hold individual log files
2. **Converted 27 existing log entries** from wiki/log.md into individual files
3. **Updated AGENTS.md** to specify:
   - ISO8601 basic format for filenames: `YYYYMMDDTHHMMSSZhhmm.md`
   - Example: `20260425T170000+0300.md`
   - Cross-platform compatible (Windows does not allow colons in filenames)
   - YAML frontmatter structure with title, type, Sources, and date-created
4. **Renamed original file** to `wiki/log.md.backup` for reference
5. **Updated Rules section** in AGENTS.md to reference `wiki/log/` instead of `wiki/log.md`

### Filename Format

Each log file uses ISO8601 basic format timestamp:
- Format: `YYYYMMDDTHHMMSSZhhmm.md`
- Example: `20260425T170000+0300.md`
- No colons or special characters (Windows compatible)
- Sortable lexicographically
- Timezone offset included

### File Structure

Each log file contains:
```markdown
---
title: YYYY-MM-DD HH:MM - DESCRIPTION OF THE CHANGE
type: log
Sources:
    - wiki/path/to/file1.md
    - wiki/path/to/file2.md
date-created: 2026-04-18T12:00:00+03:00
---

# YYYY-MM-DD HH:MM - DESCRIPTION OF THE CHANGE

Describe the change(s) and rationale

## Changed files

Add a Markdown list of the files being changed
```
